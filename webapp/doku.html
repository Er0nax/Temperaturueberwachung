API Dokumentation - Temperaturüberwachung

App Endpunkte

GET /api/web/app/download

Beschreibung: Schickt die aktuelle App Version als .zip Datei zurück.

Erforderliche Parameter: Keine

Optionale Parameter: Keine

GET /api/web/app/info

Beschreibung: Gibt die Informationen der aktuellen App zurück. Alternativ können auch Informationen einer bestimmten Version zurückgegeben werden.

Erforderliche Parameter: Keine

Optionale Parameter: version

GET /api/web/app/checkapi

Beschreibung: Gibt „true“ oder „false“ zurück, ob die API ordnungsgemäß funktioniert.

Erforderliche Parameter: Keine

Optionale Parameter: Keine

Asset Endpunkte

GET /api/web/asset/image

Beschreibung: Rendert ein Bild aus dem asset Ordner der API.

Erforderliche Parameter: src, type

Optionale Parameter: height, width

Log Endpunkte

GET /api/web/log/all

Beschreibung: Gibt alle Logs zurück. Alternativ kann nach einer Relation gefiltert werden.

Erforderliche Parameter: Keine

Optionale Parameter: relation

Role Endpunkte

GET /api/web/role/all

Beschreibung: Gibt alle Rollen mit deren Informationen zurück.

Erforderliche Parameter: Keine

Optionale Parameter: Keine

Sensor Endpunkte

GET /api/web/sensor/all

Beschreibung: Gibt alle Sensoren mit den letzten gemessenen Temperaturen zurück. Alternativ kann auch das Limit der Temperaturen geändert werden.

Erforderliche Parameter: Keine

Optionale Parameter: tempLimit

POST /api/web/sensor/simulate

Beschreibung: Erstellt jede Sekunde neue Werte für alle Sensoren. Es kann eine Dauer der Simulation angegeben werden.

Erforderliche Parameter: Keine

Optionale Parameter: loops

PUT /api/web/sensor/update

Beschreibung: Updated einen Sensor mit neuen Werten. Erfordert einen angemeldeten Benutzer mit der „Admin“ Rolle.

Erforderliche Parameter: token, sensor_id

Optionale Parameter: maxTemp, minTemp, name, color, address, active

POST /api/web/sensor/create

Beschreibung: Erstellt einen neuen Sensor. Erfordert einen angemeldeten Benutzer mit der „Admin“ Rolle.

Erforderliche Parameter: token, server_id, manufacturer_id

Optionale Parameter: maxTemp, minTemp, address, color

Server Endpunkte

GET /api/web/server/all

Beschreibung: Gibt alle Server zurück.

Erforderliche Parameter: Keine

Optionale Parameter: Keine

POST /api/web/server/create

Beschreibung: Erstellt einen neuen Server. Erfordert einen angemeldeten Benutzer mit der „Admin“ Rolle.

Erforderliche Parameter: name

Optionale Parameter: Keine

PUT /api/web/server/update

Beschreibung: Updated einen Server. Erfordert einen angemeldeten Benutzer mit der „Admin“ Rolle.

Erforderliche Parameter: server_id

Optionale Parameter: name, active

Translation Endpunkte

GET /api/web/translation/all

Beschreibung: Gibt alle Übersetzungen zurück.

Erforderliche Parameter: Keine

Optionale Parameter: Keine

GET /api/web/translation/translate

Beschreibung: Gibt einen übersetzten String zurück.

Erforderliche Parameter: t

Optionale Parameter: language, category

User Endpunkte

POST /api/web/user/login

Beschreibung: Loggt einen Benutzer ein und gibt Informationen zurück.

Erforderliche Parameter: username, password

Optionale Parameter: Keine

POST /api/web/user/register

Beschreibung: Registriert einen Benutzer und gibt die Informationen zurück.

Erforderliche Parameter: username, password, passwordRepeat

Optionale Parameter: Keine

PUT /api/web/user/update

Beschreibung: Updated einen Benutzer. Benötigt den api_token eines Users, um zu updaten.

Erforderliche Parameter: token, user_id

Optionale Parameter: username, snowflake, phone, active, imperial_system, language, avatar

GET /api/web/user/info

Beschreibung: Gibt die Informationen eines Benutzers zurück. Es kann nur einer der erforderlichen Parameter angegeben werden.

Erforderliche Parameter: id, username, snowflake, token

Optionale Parameter: Keine

POST /api/web/user/logout

Beschreibung: Loggt einen Benutzer aus. Setzt alle api_tokens des Benutzers auf inaktiv.

Erforderliche Parameter: token

Optionale Parameter: Keine

GET /api/web/user/all

Beschreibung: Gibt alle Benutzer mit deren Informationen zurück.

Erforderliche Parameter: Keine

Optionale Parameter: Keine
